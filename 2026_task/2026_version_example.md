# 2026 魔方绕桩 — 五种赛道情况说明

本目录为 **2026 年电子系统导论课程「魔方绕桩」** 任务准备，在 **[cube_v4](../cube_v4.md)** 框架上，针对五种赛道布局各提供一套**完整可运行**的参考脚本。

每个 `adapt_*` 目录内的主 `.py` 文件是**自包含**的：函数库与比赛流程写在同一个文件里，**不** `import` 仓库根目录的 `cube_v4.py`，方便拷贝到树莓派后直接运行、现场改参。

赛道尺寸与布局示意图见同目录 **[`rule.png`](./rule.png)**。

---

## 赛道与五种情况

赛道长 **7.9 m**、宽 **1.9 m**，沿长度方向设 **3 个里程碑**，各放一个彩色魔方作为绕桩目标：

| 里程碑 | 颜色 | 距左边界距离 | 是否可变 |
|--------|------|--------------|----------|
| 里程碑 1 | 蓝 | **L7** | 比赛当天随机 |
| 里程碑 2 | 黄 | **L5 = 0.87 m** | 固定居中 |
| 里程碑 3 | 红 | **L6** | 比赛当天随机 |

**L7、L6** 各自独立抽取，只能取下面三个值之一（相对赛道左边界）：

| 代号 | 距离 | 含义 |
|------|------|------|
| **L**（Left） | 0.6 m | 偏左 |
| **M**（Middle） | 0.87 m | 居中 |
| **R**（Right） | 1.14 m | 偏右 |

因此共有 **5 种**有效组合（黄块始终居中，不参与组合）：

| 目录 | 代号 | 蓝块 L7 | 红块 L6 | 说明 |
|------|------|---------|---------|------|
| `adapt_LL` | LL | L（0.6 m） | L（0.6 m） | 第一、三块均偏左 |
| `adapt_LR` | LR | L（0.6 m） | R（1.14 m） | 第一块偏左，第三块偏右 |
| `adapt_MM` | MM | M（0.87 m） | M（0.87 m） | 第一、三块均居中 |
| `adapt_RL` | RL | R（1.14 m） | L（0.6 m） | 第一块偏右，第三块偏左 |
| `adapt_RR` | RR | R（1.14 m） | R（1.14 m） | 第一、三块均偏右 |

> **命名规则**：两个字母分别表示**第一块（蓝）**与**第三块（红）**的左右位置；中间黄块位置固定，不出现在代号里。

## 目录结构

```
2026_task/
├── rule.png                    # 赛道尺寸与布局示意图
├── 2026_version_example.md     # 本说明
├── adapt_LL/                   # 情况 LL：蓝左 + 红左
│   └── LL.py                   # 主程序（函数库 + 比赛流程）
├── adapt_LR/                   # 情况 LR：蓝左 + 红右
│   └── LR.py
├── adapt_MM/                   # 情况 MM：蓝中 + 红中
│   └── MM.py
├── adapt_RL/                   # 情况 RL：蓝右 + 红左
│   └── RL.py
└── adapt_RR/                   # 情况 RR：蓝右 + 红右
    └── RR.py
```

每个 `*.py` 文件结构相同：

1. 顶部 **`Config`**：引脚、HSV、摄像头、编码器等现场参数  
2. 中部 **动作函数**：`search_color`、`approach_target`、`turn_angle`、`forward_pid` 等  
3. 底部 **`if __name__ == "__main__"`**：按该赛道情况写好的完整比赛流程（角度、直行距离、停近像素等）

不同 `adapt_*` 之间，主要差异在 **`__main__` 里的动作顺序与数值**；函数库部分基本一致。

## 运行方式

在树莓派上，先确认当天赛道属于哪一种情况，再进入对应目录运行主程序：

```bash
# 示例：当天为「蓝左 + 红左」
cd 2026_task/adapt_LL
python3 LL.py
```

其他情况同理，例如 `adapt_LR` 下运行 `python3 LR.py`。

**建议调试顺序：**

1. 先将 `Config.DRY_RUN = True`，只打印电机指令、不驱动 GPIO，检查识别与流程逻辑  
2. 再设 `DRY_RUN = False`，低速实车跑通  
3. 根据现场微调 `__main__` 中的 `turn_angle` 角度、`forward_pid` 时间、`stop_pixels` 等

HSV 颜色阈值可用仓库根目录的 **`HSV_test.py`** 临场标定。

## 与根目录库的关系

| 用途 | 推荐文件 |
|------|----------|
| 通用函数库，自行在 `main` 里拼接流程 | 根目录 [`cube_v4.py`](../cube_v4.py) 或 [`cube_v5.py`](../cube_v5.py) |
| 2026 五种赛道各一套参考满分流程 | 本目录 `adapt_*/*.py` |

说明：

- 核心 API（`search_color`、`approach_target`、`turn_angle`、`forward_pid` 等）与 **[cube_v4.md](../cube_v4.md)** 描述一致。   
- 若你已有自己的流程，只需把对应 `adapt_*` 里的 `__main__` 段落做重构，或对照修改角度与直行参数即可。

## 临场快速对照

比赛前对照 **`rule.png`** 或裁判公布的 L7、L6 数值，选目录：

```
L7 = 0.6 m  且  L6 = 0.6 m   →  adapt_LL / LL.py
L7 = 0.6 m  且  L6 = 1.14 m  →  adapt_LR / LR.py
L7 = 0.87 m 且  L6 = 0.87 m  →  adapt_MM / MM.py
L7 = 1.14 m 且  L6 = 0.6 m   →  adapt_RL / RL.py
L7 = 1.14 m 且  L6 = 1.14 m  →  adapt_RR / RR.py
```
